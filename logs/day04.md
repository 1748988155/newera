# 📚 今日学习内容

## 1. 按键状态变化检测

通过比较当前按钮状态和上一次状态，实现按键变化检测：

```cpp
if(status != lastStatus)
```

这种方式可以判断：

* 按钮被按下
* 按钮被松开

这是实现复杂按键逻辑的基础。

---

## 2. 按键按压时间计算

通过 `millis()` 记录按下时间并计算按压时长：

```cpp
pressTime = millis() - currentTime;
```

利用按压时间可以区分不同的操作：

* 短按
* 中按
* 长按

---

## 3. 实现多种按键功能

基于按键按压时间，实现了不同 LED 行为。

### 短按

短按时 LED 进行亮灭切换。

### 中按（500ms ~ 2000ms）

LED 慢速闪烁。

### 长按（>2000ms）

LED 快速闪烁。

---

## 4. 双击检测逻辑

通过记录按钮松开次数，并限制时间间隔来判断双击：

```cpp
if(millis() - openTime > 200)
{
    clickCount = 0;
}
```

如果两次点击发生在 **200ms 内**，则判定为双击。

双击触发 LED 呼吸灯效果。

---

## 5. PWM 调光实验

使用 PWM 调整 LED 亮度：

```cpp
analogWrite(led, light);
```

通过循环逐渐改变亮度模拟 **呼吸灯效果**：

```cpp
for(light = 255; light > 0; light--)
```

# ⚠ 今日遇到的问题

## 1. 程序阻塞问题

使用：

```cpp
for(...)
delay(...)
```

实现呼吸灯动画时，程序会被循环完全占用。

在循环运行期间：

* 按钮输入无法被检测
* 程序无法切换模式
* LED 状态无法被其它逻辑控制

这属于 **阻塞式代码（Blocking Code）问题**。

---

## 2. PWM 与普通 GPIO 控制冲突

在 PWM 模式运行后，有时 LED 不容易恢复到普通 `digitalWrite()` 控制。

这与 PWM 硬件控制引脚的机制有关，需要更合理的结构管理 LED 状态。

---

# 🧠 今日核心收获

今天最大的收获并不是新的语法，而是理解了一个重要概念：

**嵌入式程序应该尽量避免长时间阻塞代码。**

例如：

```cpp
delay()
for循环动画
```

在简单程序中可以使用，但在复杂交互系统中会带来很多问题。

更好的方式是：

使用 `millis()` 进行 **非阻塞定时控制**。
